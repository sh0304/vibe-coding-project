# 프로젝트 핵심 규칙 (Core Rules)

1. **이력 원본 보존 (SCD & Soft Delete)**
   - 조직/사원 데이터 변경 시 기존 레코드 수정(Update) 및 Hard Delete를 절대 금지한다.
   - `validFrom`, `validTo`를 변경하여 이력을 닫고, 새로운 레코드를 생성(Insert)하는 SCD 방식을 철저히 준수한다.

2. **데이터 스냅샷 정책**
   - 과거 기록이 중요한 행위 데이터(결재 등)는 관계형 DB 연결(JOIN)에 의존하지 않고, 행위 당시의 정보(부서명, 이름 등)를 그대로 문자열 복사(Snapshot)하여 저장한다.

3. **코딩 컨벤션 위임**
   - 구체적인 아키텍처나 코딩 작성 규칙은 `.agent/skills/coding-conventions/SKILL.md` 등 별도 스킬 파일을 따른다.

4. **AI 통제 가이드라인**
   - 어떠한 상황에서도 모든 종류의 응답(주석, 문서, 채팅)에 이모지 사용을 일절 금지한다.
   - 새로운 기능의 코드를 짤 때 항상 '과거 이력 조회에 치명적 결함이 생기지 않는가'를 최우선으로 사고한다.
