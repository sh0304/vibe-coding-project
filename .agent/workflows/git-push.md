# Git 커밋 및 원격 푸시 가이드

이 워크플로우는 작업된 최신 내용을 커밋 컨벤션에 맞춰 커밋하고, 원격 저장소(Remote)에 안전하게 푸시하기 위해 사용합니다. 푸시 실패 시의 안전장치(롤백)가 포함되어 있습니다.

1. 현재 작업 내용들의 맥락을 파악하고, `coding-conventions/SKILL.md`에 명시된 형태(`feat`, `fix` 등)를 따르되, 상세 내용은 **한글**로 커밋 메시지를 기획합니다. (예: `feat: 조직도 조회 기능 추가`)
2. 터미널에서 `git add .` 명령어를 실행하여 변경사항을 스테이징합니다.
3. 터미널에서 `git commit -m "기획한 커밋 메시지"` 명령어로 커밋을 생성합니다.
4. 터미널에서 `git push` 명령어를 실행합니다.
5. **(장애 발생 시 롤백 조치)** 만약 4번의 `git push` 과정에서 충돌(Conflict), 반려(Reject), 또는 훅(Hook) 에러 등이 발생했다면 **지체 없이 `git reset --soft HEAD~1` 명령어를 터미널에서 실행하여** 방금 한 커밋을 즉시 취소하십시오. (파일 내용은 살아있고 Staging 된 상태로 돌아갑니다.)
6. 푸시가 성공적으로 정상 완료되었는지, 혹은 에러가 발생하여 커밋을 취소했는지 현재 상황을 사용자한테 보고합니다.
