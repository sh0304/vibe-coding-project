# 통합 DB 시드 데이터 생성 (Admin + SCD History)

이 워크플로우는 시스템 초기 구동에 필요한 **관리자 기초 계정**을 생성하고, 동시에 타임머신 조회 및 비용 집계 테스트를 위한 **SCD 기반의 가상 이력 데이터**를 로컬 DB에 삽입합니다.

// turbo-all
## 1. 데이터베이스 시딩 실행
1. 관리자 권한 및 기초 데이터 생성을 위해 `npx tsx prisma/seed-admin.ts`를 실행합니다.
2. 부서 개편 및 인사 발령 등 복잡한 SCD 이력 레코드 생성을 위해 `npx tsx prisma/seed.ts`를 이어서 실행합니다.

## 2. 점검 및 보고
1. 각 스크립트 실행 결과 콘솔에 출력된 "✅ Seeding completed" 메시지를 확인합니다.
2. 실행 도중 `Unique constraint` 위반이나 스키마 불일치 에러가 발생하는지 모니터링합니다.
3. 모든 시딩이 완료되면 사용자에게 "관리자 계정과 풍부한 이력 데이터가 완벽하게 준비되었습니다"라고 보고합니다.
