# 기능 최적화 및 동기화 워크플로우

단순한 검증을 넘어, 수정 사항이 발생했을 때 **요구사항(Docs) - 코드(Source) - 화면(UI)** 간의 정합성을 맞추고 품질을 고도화하는 "Refactoring & Sync" 전용 워크플로우입니다.

## 1단계: 변경 및 불일치 탐지 (Gap Analysis)
1. 사용자의 수정 요청 사항을 분석하고, 해당 기능의 요구사항 문서(`docs/features/[기능].md`)를 읽습니다.
2. 현재 구현된 소스 코드와 문서를 대조하여 코드상으로 누락된 요건이나 설계와 다른 부분을 먼저 찾아냅니다.
3. 브라우저 서브에이전트(`browser_subagent`)를 통해 실제 화면에서 수정이 필요한 CSS, 레이아웃, 인터랙션 문제를 리스트업합니다.

## 2단계: 스마트 코드 수정 (Smart Fix)
1. 1단계에서 발견된 '불일치 리스트'를 바탕으로 코드를 수정합니다.
2. **디자인 최적화**: `tailwind-v4-shadcn` 규칙에 따라 미진한 스타일(패딩, 색상, 애니메이션)을 보강합니다.
3. **로직 안정화**: `error-handling` 및 `testing` 스킬을 적용하여 엣지 케이스 처리를 강화합니다.

## 3단계: 하이브리드 검증 (Hybrid QA)
1. **논리 검점**: 수정된 로직에 대해 `npm test`를 실행하여 기존 기능이 깨지지 않았는지(Regression Test) 확인합니다.
2. **시각 점검**: `browser_subagent`가 수정된 화면을 다시 확인하여 요구사항을 100% 충족하는지 최종 컨펌합니다.

## 4단계: 요구사항 문서 자동 동기화 (Doc Sync)
1. 코드 수정 과정에서 변경되거나 개선된 기능(예: UX 개선, 필드 추가 등)이 있다면, 이를 역으로 요구사항 문서(`docs/features/*.md`)에 반영합니다.
2. **문서가 항상 최신 구현을 보장하도록** 업데이트를 자동 수행합니다.

## 5단계: 배포 준비
1. 변경 사항을 요약하여 사용자에게 보고하고, `/git-push` 워크플로우 실행을 준비합니다.
